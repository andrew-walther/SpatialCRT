1 Executive Summary

This report is a companion to the MLE_combined_design_recommendations.pdf figure file. It provides narrative explanations of how each analysis was computed, what each figure and table shows, and what the results mean in practice for researchers designing spatial cluster randomized trials (CRTs).

The simulation evaluates 8 treatment assignment designs across 2560 parameter scenarios using Maximum Likelihood Estimation (MLE) via a spatial autoregressive model (lagsarlm). The estimand is the direct treatment effect \(\tau = 1.0\).

Three analytical questions are addressed:

Spillover type (control-only vs. both) is by far the most influential parameter for all leading designs, while spatial autocorrelation \(\rho\) and spillover magnitude \(\gamma\) have secondary effects.

2 Study Background

2.1 Designs Evaluated

8 treatment assignment strategies are compared. Each allocates approximately 50% of the 100 clusters (\(10 \times 10\) grid) to treatment:

8 treatment assignment designs evaluated. Deterministic designs (1, 2) produce one assignment and replicate it; stochastic designs resample across 25 replications.
ID Name Deterministic? Description
1 Checkerboard Yes Alternating grid pattern; maximizes spatial separation of treated and control units
2 High Incidence Focus Yes Treats the top-50\% incidence clusters deterministically
3 Saturation Quadrants No Assigns random saturation levels (0.2--0.8) within each of four spatial quadrants
4 Isolation Buffer No Greedy assignment ensuring no two adjacent clusters are both treated
5 2x2 Blocking No 1:1 randomization within non-overlapping 2x2 spatial blocks
6 Balanced Quartiles No Stratified randomization balanced within incidence quartiles
7 Balanced Halves No Stratified randomization balanced within incidence halves (above/below median)
8 Incidence-Guided Saturation Quadrants No NA

2.2 Performance Metrics

For each of the 2560 scenarios, 250 simulation iterations (25 design resamples \(\times\) 10 outcome resamples) produce the following metrics:

  • Bias \(= \bar{\hat{\tau}} - \tau\): average departure from true effect \(\tau = 1.0\)
  • SD: standard deviation of estimates across iterations
  • MSE \(= \text{Bias}^2 + \text{SD}^2\): total estimation error; the primary ranking criterion throughout this report
  • Coverage: fraction of 95% confidence intervals containing \(\tau = 1.0\)

Lower MSE is better. Coverage near 0.95 (the nominal level) indicates valid inference.

3 Analytical Methods

This section explains exactly how each analysis in the recommendation report is computed.

3.1 Core Ranking Method: rank_designs_by_group()

All conditional rankings in this report share the same logic, implemented in rank_designs_by_group():

  1. Group the 512 scenarios within an incidence config by the requested conditioning variables plus Design (e.g., group by {Rho, Design})
  2. Aggregate: compute average MSE, Coverage, Bias, and SD within each group
  3. Rank: within each unique combination of the conditioning variables, assign integer ranks to designs based on ascending average MSE (rank 1 = best; ties broken using minimum rank)
  4. Return a tidy data frame with one row per {group, Design} combination

This is the same ranking logic used in table_design_ranks() in 06_visualizations.R, extended to support arbitrary conditioning sets.

Critical invariant: Rankings are always computed within a single incidence configuration. Results are never aggregated across the 5 incidence configs because each represents a fundamentally different assumption about how prior incidence data arises.

3.2 Q1 — Per-Config Rankings: table_incidence_rankings() and plot_incidence_rankings()

What is computed: For each of the 5 incidence configs, all 512 scenarios (8 designs \(\times\) 64 parameter combinations) are summarized by computing the average MSE per design. Designs are then ranked 1–8 (lowest = best) within each config.

What the figure shows (Page 1 of PDF): A faceted bar chart with one panel per incidence config. Bars represent average MSE per design, sorted from lowest to highest within each panel. Rank labels (1–8) appear above each bar.

How to read it: Lower bars are better. Rank labels allow quick comparison across panels. A design ranked consistently high across all panels is a robust global recommendation.

3.3 Q2 — Marginal Rankings: table_marginal_rankings() and plot_rank_trajectories()

What “marginal” means: For a parameter like Rho, we want to know: ignoring all other parameters, how does each design rank when Rho takes a specific value? This is computed by grouping all scenarios by {Rho, Design} and averaging MSE across the remaining parameters (Gamma, Spillover_Type, Neighbor_Type).

What the rank trajectory plot shows: A line plot where the x-axis is the parameter level, the y-axis is design rank (1 at top, 8 at bottom), and each line traces one design. Crossing lines indicate a rank reversal: a design that is best at one parameter level but not at another.

What the marginal ranking table shows: A wide table with parameter levels as column headers and designs as rows. Each cell reports Avg_MSE (#Rank) at that parameter level, averaged over all other parameters. Designs are ordered by their overall rank (best at top).

What the conditional MSE boxplot shows: Rather than averages, this shows the full distribution of raw scenario-level MSE values for each design at each parameter level. This reveals whether a design’s advantage is consistent or comes from a few favorable extreme scenarios.

3.4 Q3 — Best Design Heatmap: plot_best_design_heatmap()

What is computed: For every combination of two parameters (e.g., Rho and Gamma), all scenarios sharing that combination are grouped, average MSE is computed per design, and the design with the minimum average MSE is identified as the winner.

What the heatmap shows: A grid of colored tiles where each cell corresponds to one {Rho, Gamma} combination. The color and text label identify the winning design. The MSE value inside each tile is the winning design’s average MSE at that combination.

Interpretation guidance: If one design wins all tiles, it dominates unconditionally. If tiles show different winners, the choice of design should adapt to the anticipated parameter regime.

3.5 Programmatic Commentary: generate_commentary()

For each incidence config, six structured findings are automatically derived from the results:

Finding Computed as
Overall winner Design with Rank 1 from ungrouped rank_designs_by_group()
Dominance % From scenario-level pairwise comparison: \(\frac{\#\{s : \text{MSE}_{\text{best},s} < \text{MSE}_{\text{2nd},s}\}}{64}\)
Rank stability Whether the best design holds rank 1 at every level of Rho and every level of Gamma
Coverage Average Coverage for the best and worst designs
Sensitivity Eta-squared (\(\eta^2 = SS_{\text{between}} / SS_{\text{total}}\)) for each parameter
Recommendation Template string synthesizing findings 1–5

4.1 Cross-Config MSE Rankings

Reading the plot: The 5 panels correspond to the 5 incidence configurations. Within each panel, designs are arranged from best (left) to worst (right), with the rank number printed above each bar. The consistent leftward position of D3: Sat. Quadrants across all panels is the primary finding.

Key observation: D3: Sat. Quadrants ranks first in 2 of 5 incidence configurations, with average MSE ranging from 0.073 to 0.08 depending on the config. This 8.3–13.6x MSE advantage over the worst design (D1: Checkerboard) is stable regardless of how the historical incidence was generated.

4.2 Cross-Config Coverage Rankings

Reading the plot: The red dashed line at 0.95 is the nominal coverage target. Bars above this line are over-covering (conservative CIs); bars below are under-covering (anti-conservative). Under MLE, designs other than D1: Checkerboard all achieve reasonable coverage (0.91–0.97), but D1: Checkerboard falls dramatically short.

Key observation: D1: Checkerboard’s poor coverage (0.51–0.58) makes it unsuitable for inference regardless of MSE. Even in the rare scenario where its MSE is competitive, its confidence intervals are invalid.

5 Q2: Marginal Rankings by Parameter Level

This section answers: holding all other parameters at their average, how does each design’s rank change as one parameter varies?

5.1 Effect of Outcome Spatial Autocorrelation (Rho)

How to read the trajectory plots: Rank 1 sits at the top; rank 8 at the bottom. A flat horizontal line means a design’s rank does not change as Rho increases. A downward slope means the design falls in rank as Rho increases. Crossing lines between two designs indicate a rank reversal.

Findings:

  • D3: Sat. Quadrants is largely stable across Rho levels, with exceptions: iid Uniform at Rho=0.01,0.5; Poisson (rho_X = 0.20) at Rho=0.01,0.2,0.5; Poisson (rho_X = 0.50) at Rho=0,0.2; Spatial (rho_X = 0.20) at Rho=0,0.5; Spatial (rho_X = 0.50) at Rho=0,0.01,0.2,0.5.
  • D1: Checkerboard consistently occupies last rank across all Rho values, confirming its failure is not a high-Rho artifact.

5.2 Effect of Spillover Magnitude (Gamma)

Findings: Gamma (spillover magnitude) has virtually no effect on design rankings. All 8 designs maintain their relative ranks across Gamma = 0.5 to 0.8 within every incidence config. This is consistent with the sensitivity analysis finding that Gamma explains less than 1% of MSE variance (eta-squared \(<\) 0.002) for all designs. Researchers need not tailor design selection to anticipated spillover magnitude.

9.2 Why D3: Sat. Quadrants Wins: Mechanistic Explanation

D3: Sat. Quadrants works by dividing the \(10 \times 10\) grid into four \(5 \times 5\) quadrants, where each quadrant receives a random treatment saturation level drawn from a discrete set (0.20, 0.40, 0.60, 0.80). Within each quadrant, clusters are randomly selected for treatment at the assigned saturation rate.

This design outperforms others because:

  1. Quadrant-level saturation variation creates spatial contrast. The MLE estimator (lagsarlm) must disentangle the direct treatment effect \(\tau\) from the spatial lag parameter \(\rho\). When treatment is uniformly distributed or strongly correlated with incidence, this separation is harder. Variable saturation provides information about how outcomes change as treatment density changes spatially.

  2. Randomness within quadrants prevents systematic confounding. Unlike deterministic designs, idiosyncratic variation uncorrelated with spatially-structured incidence or outcome patterns is introduced.

  3. Quadrant-level structure respects spatial scale. The \(5 \times 5\) quadrant blocks match the scale at which spatial autocorrelation operates (rook/queen neighborhoods), providing richer contrast than finer (2x2) or coarser structures.

9.3 Why D1: Checkerboard Fails: Mechanistic Explanation

D1: Checkerboard assigns treatment in a perfectly alternating pattern. This creates a fundamental identification problem:

  • The treatment assignment vector \(Z\) is a deterministic function of position on the grid.
  • The spatial weight matrix \(W\) encodes neighbor relationships, which are also determined by position.
  • Therefore, \(Z\) and \(W \cdot Z\) (the spillover term) are collinear with the spatial lag of the outcome \(W \cdot Y\) that the SAR model needs to estimate \(\rho\).

This collinearity inflates variance, biases \(\hat{\tau}\), and degrades coverage — all of which are observed in the simulation results.

Conclusions:

  1. Universal recommendation: Under MLE estimation, D8: Inc.-Guided Sat. Q. is the best treatment assignment design across all incidence modes, Rho levels, and Gamma levels tested. It achieves 9.4x lower MSE than the worst design.

  2. Spillover type is the key moderator. When spillover direction is “both”, D6: Balanced Quartiles becomes competitive and should be considered. When spillover is “control_only”, D3: Sat. Quadrants’s advantage is unambiguous (50% of tiles).

  5. Coverage is near-nominal for leading designs. Under MLE, the top 4 designs all achieve 0.94–0.94% coverage across the tested scenarios, confirming that MLE-based inference is valid when a well-chosen spatial design is used.
